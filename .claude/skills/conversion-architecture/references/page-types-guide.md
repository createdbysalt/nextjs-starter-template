# Page Types Guide

## Overview

Every page type has a specific job. This guide details the purpose, structure, and best practices for common page types.

---

## Homepage

### Job Statement
"Orient visitors, build initial trust, and route them to the right path based on their needs."

### Audience
- Mixed awareness levels
- First impression for many
- Returning visitors seeking specific content

### Required Elements
| Element | Purpose |
|---------|---------|
| Hero with value prop | Immediately communicate what you do and for whom |
| Primary CTA | Clear next step for ready visitors |
| Navigation | Wayfinding to key sections |
| Trust indicators | Logos, testimonials, credentials (above fold if possible) |
| Key benefits/services | What do you offer? |
| Social proof | Why should they trust you? |
| Secondary paths | For visitors who aren't ready for primary CTA |

### Structure Pattern
```
1. Hero (Value prop + CTA)
2. Trust bar (Logos, credentials)
3. What we do / How it works
4. Key benefits or features
5. Social proof (Testimonials)
6. Secondary content (Segment routing, use cases)
7. Final CTA
```

### Common Mistakes
- Trying to say everything (cluttered)
- No clear CTA (directionless)
- Generic messaging (forgettable)
- Missing trust elements (unconvincing)

## Product/Service Page

### Job Statement
"Explain the offering clearly and convert interested visitors into customers/leads."

### Audience
- Product-aware visitors
- People evaluating this specific offering
- Comparison shoppers

### Required Elements
| Element | Purpose |
|---------|---------|
| Clear name and description | What is it? |
| Key benefits | Why should they care? |
| Features/inclusions | What do they get? |
| Pricing (or link to pricing) | How much? |
| Social proof | Does it work? |
| CTA | Buy/signup/contact |
| Objection handling | Address hesitations |
| FAQs | Answer common questions |

### Structure Pattern
```
1. Hero (Product name + primary benefit + CTA)
2. Problem/Solution context
3. Key benefits (3-5)
4. How it works / Features
5. Social proof (Testimonials specific to this product)
6. Pricing or pricing CTA
7. FAQ
8. Final CTA
```

### Common Mistakes
- Feature-focused instead of benefit-focused
- No social proof for this specific product
- CTA only at bottom (make it accessible throughout)
- Missing pricing or vague pricing

## Pricing Page

### Job Statement
"Make the decision easy by clearly presenting options and reducing friction to purchase."

### Audience
- Product-aware, close to decision
- Comparing you to alternatives
- Budget-conscious evaluators

### Required Elements
| Element | Purpose |
|---------|---------|
| Clear pricing tiers | What are the options? |
| What's included | Feature comparison |
| Recommended tier | Help them decide |
| CTA for each tier | Easy to act |
| Trust elements | Reduce risk (guarantee, testimonials) |
| FAQ | Handle price objections |

### Structure Pattern
```
1. Simple headline (e.g., "Simple, transparent pricing")
2. Pricing cards (side by side)
3. Feature comparison table (if complex)
4. Trust elements (guarantee, security badges)
5. Testimonials about value
6. FAQ (price objections)
7. CTA / Contact for questions
```

### Pricing Display Best Practices
- Highlight recommended tier
- Show monthly AND annual (if applicable)
- Use specific numbers ($49, not "from $49")
- Anchor with higher tier first (or highlight middle)
- Include social proof near price

### Common Mistakes
- Too many tiers (confusing)
- Hidden costs revealed later (trust-breaking)
- No recommendation (decision paralysis)
- No risk reduction (guarantee, trial)

## About Page

### Job Statement
"Build trust and human connection by showing who you are and why you do what you do."

### Audience
- Evaluating visitors who want to know the company
- People checking legitimacy
- Culture/values alignment seekers

### Required Elements
| Element | Purpose |
|---------|---------|
| Company story | Origin, mission, why |
| Team (or founder) | Human faces |
| Values/beliefs | What you stand for |
| Credibility markers | Awards, press, credentials |
| CTA | Don't leave them hanging |

### Structure Pattern
```
1. Hero (Mission statement or story hook)
2. Our story / Why we exist
3. What we believe / Our approach
4. Team section (photos, bios)
5. Credibility (press, awards, credentials)
6. CTA (Join us, Work with us, etc.)
```

### Common Mistakes
- Boring corporate speak (unrelatable)
- No humans shown (faceless company)
- Just a timeline (who cares when you were founded?)
- No CTA (dead end)

## Case Study Page

### Job Statement
"Prove your solution works by showing specific, relatable success stories."

### Audience
- Product-aware, seeking proof
- Skeptical evaluators
- People looking for relevance ("Is this for someone like me?")

### Required Elements
| Element | Purpose |
|---------|---------|
| Client identification | Who is this about? |
| Challenge | What problem did they have? |
| Solution | What did you do? |
| Results | What happened? (Specific!) |
| Testimonial quote | Their words |
| CTA | "Want results like this?" |

### Structure Pattern (Classic)
```
1. Hero (Client name + headline result)
2. About the client (context)
3. The challenge (before state)
4. The solution (what you did)
5. The results (after state - specific numbers!)
6. Testimonial quote
7. CTA
```

### Structure Pattern (Narrative)
```
1. The situation (set the scene)
2. The struggle (build tension)
3. The turning point (entering your solution)
4. The transformation (results)
5. The quote (in their words)
6. The invitation (CTA)
```

### Common Mistakes
- Vague results ("Improved efficiency")
- No numbers or specifics
- Client unrelatable to target audience
- No permission / attribution
- Missing CTA

## Blog Post / Article

### Job Statement
"Attract and educate visitors, moving them from unaware to problem/solution-aware."

### Audience
- Unaware or problem-aware
- Information seekers
- SEO traffic

### Required Elements
| Element | Purpose |
|---------|---------|
| Compelling headline | Stop the scroll |
| Valuable content | Actually help them |
| Internal links | Keep them on site |
| CTA / Next step | Don't let them leave empty |
| Author info | Build trust |
| Related posts | Further engagement |

### Structure Pattern
```
1. Headline (Hook + Benefit)
2. Introduction (Promise + Relevance)
3. Body content (Organized, scannable)
4. Key takeaways / Summary
5. CTA (Related offer, newsletter, next post)
6. Author bio
7. Related posts
```

### Common Mistakes
- No CTA (traffic wasted)
- Not linking to product/service (missed conversion)
- Thin content (no value)
- Keyword stuffing (bad UX)
- No internal linking (SEO waste)

## Landing Page (Campaign/Ads)

### Job Statement
"Convert paid traffic by delivering on the ad promise and driving ONE action."

### Audience
- Specific segment targeted by campaign
- Single awareness level (matches ad)
- Already interested (they clicked)

### Required Elements
| Element | Purpose |
|---------|---------|
| Message match | Deliver what ad promised |
| Single CTA | One action, one goal |
| No navigation | Prevent escape |
| Compelling headline | Continue the conversation |
| Benefits | Why they should care |
| Proof | Why they should believe |
| Urgency (if appropriate) | Why act now |

### Structure Pattern
```
1. Hero (Message match headline + CTA)
2. Pain point acknowledgment
3. Solution introduction
4. Benefits (3-5 key points)
5. Social proof
6. CTA (repeated)
7. Objection handling / FAQ
8. Final CTA with urgency
```

### Common Mistakes
- Navigation present (leaky page)
- Multiple CTAs competing
- Message mismatch with ad
- Too long without CTA
- No proof elements

## Contact Page

### Job Statement
"Make it easy to get in touch and capture leads with minimal friction."

### Audience
- Ready to take action
- Have questions
- Need human interaction

### Required Elements
| Element | Purpose |
|---------|---------|
| Contact form | Primary capture |
| Alternative contact methods | Phone, email for preferences |
| Expectation setting | When will they hear back? |
| Location (if relevant) | Physical presence |
| Map (if local) | Help them find you |

### Structure Pattern
```
1. Simple headline ("Let's talk", "Get in touch")
2. Brief context (What happens when they reach out?)
3. Contact form (minimal fields!)
4. Alternative contact methods
5. Location / Hours (if applicable)
6. FAQ or common questions
```

### Form Best Practices
- Minimum viable fields (more fields = fewer conversions)
- Clear labels
- Mobile-friendly
- Success state / confirmation
- Error handling

### Common Mistakes
- Too many form fields
- No confirmation message
- No response time expectation
- Form only (no phone/email alternative)
- Buried in footer

## FAQ Page

### Job Statement
"Answer common questions and handle objections proactively."

### Audience
- Evaluating visitors with specific questions
- People with objections
- Those who prefer self-service

### Required Elements
| Element | Purpose |
|---------|---------|
| Questions in customer language | Match how they ask |
| Clear, concise answers | Actually answer |
| Organization by topic | Easy to find |
| Search (if many FAQs) | Quick access |
| Contact option | For unanswered questions |

### Structure Pattern
```
1. Headline
2. Category navigation (if many questions)
3. Expandable Q&A format
4. Contact CTA (Can't find your answer?)
```

### What Questions to Include
1. Questions sales team hears repeatedly
2. Questions from support tickets
3. Questions that prevent conversion (objections)
4. Questions from chat logs
5. Questions competitors' FAQs answer

### Common Mistakes
- Internal jargon in questions
- Answers that don't actually answer
- No categorization (overwhelming)
- Missing the objection-handling opportunity
- No way to ask questions not listed
