# User Flow Examples

## How to Read These Examples

Each flow shows:
- The target audience and their awareness level
- The entry point (how they arrive)
- The step-by-step path with user psychology
- Friction points and mitigation strategies
- Alternative paths for non-converters

---

## Flow 1: B2B SaaS - Demo Request

### Context
**Product:** Project management software
**ICP:** Marketing managers at agencies (25-100 employees)
**Awareness:** Solution-aware (comparing tools)
**Entry:** Google search "agency project management software"

### The Flow

```
┌─────────────────────────────────────────────────────────────┐
│ STEP 1: Landing Page (SEO-optimized for search term)        │
├─────────────────────────────────────────────────────────────┤
│ User arrives thinking: "Let me see if this fits agencies"   │
│ User needs: Relevance confirmation, credibility              │
│ Page shows: Agency-specific headline, agency logos,          │
│             benefits for agencies                            │
│ User clicks: "See Features" or "See Pricing"                 │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ STEP 2: Features Page                                        │
├─────────────────────────────────────────────────────────────┤
│ User thinking: "Does it have what I need?"                   │
│ User needs: Feature confirmation, differentiation            │
│ Page shows: Features organized by agency use case,           │
│             comparison to alternatives                       │
│ User clicks: "View Case Studies" (needs proof)               │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ STEP 3: Case Study (Agency Success Story)                    │
├─────────────────────────────────────────────────────────────┤
│ User thinking: "Has this worked for agencies like mine?"     │
│ User needs: Relatable proof, specific results                │
│ Page shows: Similar agency, their challenge, results,        │
│             testimonial                                      │
│ User clicks: "Get a Demo" (convinced enough to talk)         │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ STEP 4: Demo Request Form                                    │
├─────────────────────────────────────────────────────────────┤
│ User thinking: "Let me see it in action"                     │
│ User needs: Low friction, clear expectations                 │
│ Page shows: Simple form, "30-min personalized demo",         │
│             no credit card needed                            │
│ User converts: Submits form                                  │
└─────────────────────────────────────────────────────────────┘
```

### Friction Points & Mitigation

| Friction Point | Mitigation |
|----------------|------------|
| "Is this for agencies?" | Agency-specific landing page, logos |
| "Does it have X feature?" | Feature comparison, searchable features |
| "Will it work for us?" | Agency case study, similar size company |
| "Is the demo high pressure?" | "No obligation" language, clear time commitment |

### Alternative Path (Not Ready for Demo)

```
If not ready → Offer: "Download Agency Project Management Guide"
→ Email nurture sequence
→ Retarget with case study content
→ Re-engage with demo offer
```

### Friction Points & Mitigation

| Friction Point | Mitigation |
|----------------|------------|

| "Which product is right for me?" | Skin quiz, routine builder |
| "What if it doesn't work?" | Money-back guarantee, free returns |
| "Is shipping expensive?" | Free shipping threshold, show in cart |
| "Checkout too complicated" | Express checkout, minimal fields |

### Alternative Path (Abandonment Recovery)

```
If abandons cart → Email: "Forgot something?" + 10% off
If abandons after quiz → Email: "Your personalized routine is waiting"
If browses but no add → Retarget with review content
```

## Flow 3: Professional Services - Consultation

### Context
**Product:** Financial planning services
**ICP:** High-income professionals, 35-55
**Awareness:** Problem-aware (knows they need help, unsure who)
**Entry:** Google search "financial planner for tech executives"

### The Flow

```
┌─────────────────────────────────────────────────────────────┐
│ STEP 1: Service Page (SEO-optimized)                         │
├─────────────────────────────────────────────────────────────┤
│ User arrives thinking: "Do they understand tech exec needs?" │
│ User needs: Relevance, credibility, expertise signals        │
│ Page shows: "Financial Planning for Tech Leaders" headline,  │
│             specific challenges (RSUs, IPO planning, etc.)   │
│ User clicks: "See Our Process" or "Meet Our Team"            │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ STEP 2: Our Approach Page                                    │
├─────────────────────────────────────────────────────────────┤
│ User thinking: "What makes them different?"                  │
│ User needs: Differentiation, trust in methodology            │
│ Page shows: Proprietary process, credentials, philosophy,    │
│             what to expect                                   │
│ User clicks: "View Client Results" (needs proof)             │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ STEP 3: Client Testimonials / Case Studies                   │
├─────────────────────────────────────────────────────────────┤
│ User thinking: "Have they helped people like me?"            │
│ User needs: Relatable success stories, specific outcomes     │
│ Page shows: Client testimonials (tech exec profiles),        │
│             specific outcomes (not just "great service")     │
│ User clicks: "Schedule Consultation" (ready to talk)         │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ STEP 4: Consultation Scheduling                              │
├─────────────────────────────────────────────────────────────┤
│ User thinking: "Will this be a sales pitch?"                 │
│ User needs: Low commitment, clear expectations, easy booking │
│ Page shows: "Complimentary 30-min strategy session",         │
│             Calendly embed, "No obligation" language         │
│ User converts: Books consultation                            │
└─────────────────────────────────────────────────────────────┘
```

### Friction Points & Mitigation

| Friction Point | Mitigation |
|----------------|------------|
| "Are they qualified?" | Credentials, certifications prominent |
| "Do they get my situation?" | Tech-exec specific language, case studies |
| "Is this a sales trap?" | "Strategy session" not "sales call", no obligation |
| "Scheduling is annoying" | Calendly integration, pick your time |

### Alternative Path (High Consideration)

```
If not ready → Offer: "Download Tech Executive Financial Planning Guide"
→ Email nurture with educational content
→ Invite to webinar on RSU strategies
→ Follow up with consultation offer
```

## Flow 4: SaaS - Free Trial Conversion

### Context
**Product:** Email marketing tool
**ICP:** Small business owners
**Awareness:** Product-aware (knows your tool, evaluating)
**Entry:** Direct/branded search

### The Flow

```
┌─────────────────────────────────────────────────────────────┐
│ STEP 1: Homepage                                             │
├─────────────────────────────────────────────────────────────┤
│ User arrives thinking: "Let me see what this tool can do"    │
│ User needs: Quick overview, easy path to try                 │
│ Page shows: Clear value prop, "Start Free Trial" prominent,  │
│             quick features overview                          │
│ User clicks: "Start Free Trial" (ready to try)               │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ STEP 2: Signup Form                                          │
├─────────────────────────────────────────────────────────────┤
│ User thinking: "I hope this is easy"                         │
│ User needs: Minimal friction, no credit card                 │
│ Page shows: Email + password only, "No credit card needed",  │
│             Google/social signup option                      │
│ User action: Creates account                                 │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ STEP 3: Onboarding Flow                                      │
├─────────────────────────────────────────────────────────────┤
│ User thinking: "How do I get started?"                       │
│ User needs: Quick win, guided setup                          │
│ Page shows: Progress bar, "Send your first email in 5 min",  │
│             template selection, guided wizard                │
│ User action: Completes first email setup                     │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ STEP 4: First Success (Aha Moment)                           │
├─────────────────────────────────────────────────────────────┤
│ User thinking: "This actually works!"                        │
│ User experience: Sends first email, sees open rates          │
│ System shows: Celebration moment, "You did it!",             │
│               nudge toward next feature                      │
│ User action: Continues exploring                             │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ STEP 5: Upgrade Prompt (at value moment)                     │
├─────────────────────────────────────────────────────────────┤
│ User thinking: "I could use more features/contacts"          │
│ User needs: Clear upgrade value, easy upgrade                │
│ Shows: "Unlock automation" at moment they need it,           │
│        clear pricing, seamless upgrade                       │
│ User converts: Upgrades to paid                              │
└─────────────────────────────────────────────────────────────┘
```

### Friction Points & Mitigation

| Friction Point | Mitigation |
|----------------|------------|
| "Do I need a credit card?" | "No credit card required" prominent |
| "This looks complicated" | Guided onboarding, templates |
| "I don't have time to learn" | "Send first email in 5 min" promise |
| "Why should I upgrade?" | Show upgrade at moment of need, not randomly |

### Alternative Path (Trial Abandonment)

```
If doesn't complete onboarding → Email: "Need help getting started?"
If doesn't send first email → In-app prompt + email with template
If trial expiring without conversion → Email with extension offer
If churns after trial → Win-back email sequence
```
