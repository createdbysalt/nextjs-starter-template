## Calibration Questions

When assigning severity, ask:

### For CRITICAL consideration:
- Would you delay launch to fix this?
- Is the primary conversion path affected?
- Would this cause immediate bounce or abandonment?
- Is there a fundamental trust or clarity issue?

### For HIGH consideration:
- Does research/data show this hurts conversion?
- Is a known ICP objection going unaddressed?
- Is there significant friction at a key moment?
- Would you prioritize this in the first sprint post-launch?

### For MEDIUM consideration:
- Is this suboptimal but not blocking?
- Could this be better based on best practices?
- Is this secondary to the primary conversion path?
- Would you fix this in the second or third sprint?

### For LOW consideration:
- Is this a nice-to-have polish item?
- Would you A/B test this rather than just implement?
- Is the impact likely marginal?
- Is this an edge case or minor UX improvement?

## Common Calibration Mistakes

### Over-Criticaling
```
Mistake: Marking generic CTA copy as CRITICAL
Reality: CTAs exist and work - it's HIGH at most
Rule: CRITICAL means broken or impossible, not suboptimal
```

### Under-Severitying
```
Mistake: Marking "no value proposition" as MEDIUM
Reality: Visitors can't know if they're in the right place
Rule: Clarity issues are usually HIGH or CRITICAL
```

### Over-Nitpicking
```
Mistake: 50 issues, half are button colors or font sizes
Reality: These distract from real problems
Rule: If impact is marginal, it's LOW or don't include
```

### Missing Context
```
Mistake: Same severity for every "missing testimonial"
Reality: Missing on homepage is HIGH, missing on FAQ is LOW
Rule: Context determines impact determines severity
```
